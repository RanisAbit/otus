# Проектная работа «Восстановление инфраструктуры с нуля»

## Описание проекта

Данный проект предназначен для поэтапного развёртывания учебной инфраструктуры с нуля на базе Ubuntu Server и Docker.

В состав инфраструктуры входят:
- backend-серверы с MySQL master/slave и WordPress;
- proxy-сервер с Nginx;
- сервер мониторинга с Prometheus и Grafana;
- сервер логирования с Elasticsearch, Filebeat, Logstash и Kibana.

## Состав проекта

- **Backend** — MySQL master/slave в Docker, WordPress
- **Proxy** — Nginx в Docker
- **Monitoring** — Prometheus + Grafana в Docker
- **Logging** — Elasticsearch / Filebeat / Logstash / Kibana в Docker

## Требования

Для развёртывания проекта необходимы:

- ОС Ubuntu Server 24.04.4
- SSH-доступ до виртуальных машин по 22 порту
- Доступ в интернет для скачивания пакетов, образов Docker и конфигурационных файлов
- Sudo-права для пользователя, от имени которого выполняется установка
- Открытые порты для работы сервисов

### Используемые порты

- **22** — SSH
- **80** — Nginx
- **8080** — backend / WordPress
- **3306** — MySQL
- **9100** — Node Exporter
- **9090** — Prometheus
- **3000** — Grafana
- **5044** — Logstash
- **5601** — Kibana
- **9200** — Elasticsearch

## Параметры виртуальных машин

Перед началом развёртывания необходимо подготовить виртуальные машины со следующими параметрами:

| Роль сервера | Hostname | CPU | RAM | HDD |
|---|---|---:|---:|---:|
| Proxy | `proxy` | 2 | 4 GB | 10 GB |
| Backend-1 | `backend-1` | 2 | 4 GB | 10 GB |
| Backend-2 | `backend-2` | 2 | 4 GB | 10 GB |
| Monitoring + Logging | `observ` | 4 | 6 GB | 20 GB |

## Порядок развёртывания

Развёртывание инфраструктуры рекомендуется выполнять в следующем порядке:

1. Подготовить виртуальные машины.
2. Развернуть backend-серверы.
3. Развернуть proxy-сервер.
4. Установить агенты мониторинга и логирования.
5. Развернуть monitoring-сервер.
6. Развернуть logging-сервер.
7. Проверить доступность всех сервисов.
