# Инструкция по развертыванию сервера логирования

Данный документ описывает порядок развертывания сервера логирования инфраструктуры проекта.

В результате выполнения инструкции должны быть настроены:
- Elasticsearch в Docker-контейнере;
- Logstash в Docker-контейнере;
- Kibana в Docker-контейнере.

## Состав сервера логирования

Сервер логирования включает в себя:
- Docker для развертывания Elasticsearch, Logstash и Kibana.

## Используемые файлы

### Скрипты
- `script/logging.sh` — установка Elasticsearch, Logstash и Kibana в Docker.

### Docker Compose
- `docker/elk/elasticsearch/docker-compose.yml`
- `docker/elk/logstash/docker-compose.yml`
- `docker/elk/kibana/docker-compose.yml`

### Конфигурационные файлы
- `docker/elk/logstash/config/logstash.yml`
- `docker/elk/logstash/config/pipelines.yml`
- `docker/elk/logstash/pipeline/logstash.conf`
- `docker/elk/kibana/config/kibana.yml`

## Предварительные требования

Перед началом развертывания необходимо убедиться, что:
- сервер `logging` доступен по SSH;
- на сервере используется Ubuntu Server;
- сервер имеет доступ в интернет;
- доступен пользователь `root` или пользователь с правами `sudo`.

Во всех примерах ниже предполагается выполнение команд от имени `root`.

## Таблица развертывания

| Сервер | Назначение | Используемые скрипты |
|---|---|---|
| `observ` | Monitoring + Logging | `monitoring.sh`, `logging.sh` |

## Общая схема запуска скриптов

Для каждого этапа используется одинаковый порядок действий:
1. Подключиться к серверу по SSH.
2. Скачать необходимый скрипт из репозитория с помощью `curl`.
3. Сделать файл исполняемым.
4. Запустить скрипт.
5. Проверить результат выполнения.

Если скрипт не удалось скачать автоматически, его можно создать вручную и вставить в него содержимое из репозитория.

## Порядок развертывания

### Установка сервера логирования

1. Скачать скрипт:

```bash
curl -O https://raw.githubusercontent.com/RanisAbit/otus/refs/heads/main/script/logging.sh
```

2. Сделать файл исполняемым:

```bash
chmod +x logging.sh
```

3. Запустить скрипт:

```bash
./logging.sh
```

4. Во время выполнения скрипта не требуется вручную вводить IP-адреса, так как адрес сервера определяется автоматически.

## Проверка работоспособности

После завершения всех этапов необходимо убедиться, что:
- Elasticsearch доступен по порту `9200`;
- Logstash принимает события по порту `5044`;
- Kibana доступна по порту `5601`.

## Команды для проверки

Проверка сервиса Elasticsearch:

```bash
docker ps | grep elasticsearch
curl http://localhost:9200
```

Проверка сервиса Logstash:

```bash
docker ps | grep logstash
```

Проверка сервиса Kibana:

```bash
docker ps | grep kibana
```

Проверка отправки событий на сервер Logstash:

```bash
tcpdump 'host "IP_Адрес_сервера" and port 5044'
```

## Примечания

- Скрипт автоматически устанавливает Docker, если он отсутствует.
- Скрипт создаёт необходимые каталоги и конфигурационные файлы через heredoc (`EOF`).
- При изменении логики развертывания необходимо вносить изменения не только в отдельные конфигурационные файлы репозитория, но и в соответствующие скрипты.
- Скрипт определяет IP-адрес сервера автоматически и использует его в конфигурации Logstash и Kibana.
- Перед запуском скрипта рекомендуется убедиться, что значение `vm.max_map_count` соответствует требованиям Elasticsearch.
