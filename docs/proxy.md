# Инструкция по развертыванию proxy-сервера

Данный документ описывает порядок развертывания proxy-сервера в составе инфраструктуры проекта.

В результате выполнения инструкции должны быть настроены:
- Docker для развертывания proxy-сервера;
- Nginx в контейнере Docker;
- node_exporter для передачи метрик в Prometheus;
- Filebeat для передачи логов в Logstash.

## Состав proxy-сервера

Proxy-сервер включает в себя:
- Docker для развертывания Nginx;

## Используемые файлы

### Скрипты
- `scripts/proxy.sh` — развертывание Docker-контейнера с Nginx и настройка балансировки между backend-серверами;


### Docker Compose
- `docker/nginx/docker-compose.yml`

### Конфигурационные файлы
- `configs/nginx/nginx.conf`

## Предварительные требования

Перед началом развертывания необходимо убедиться, что:
- сервер `proxy` доступен по SSH;
- на сервере используется Ubuntu Server;
- сервер имеет доступ в интернет;
- известны IP-адреса `backend-1` и `backend-2`;
- доступен пользователь `root` или пользователь с правами `sudo`.

Во всех примерах ниже предполагается выполнение команд от имени `root`.

## Таблица развертывания

| Сервер | Назначение | Используемые скрипты |
|---|---|---|
| `proxy` | Nginx reverse proxy, балансировка | `proxy.sh` |

## Общая схема запуска скриптов

Для каждого этапа используется одинаковый порядок действий:
1. Подключиться к серверу по SSH.
2. Скачать необходимый скрипт из репозитория с помощью `curl`.
3. Сделать файл исполняемым.
4. Запустить скрипт.
5. Проверить результат выполнения.

Если скрипт не удалось скачать автоматически, его можно создать вручную и вставить в него содержимое из репозитория.

## Порядок развертывания

### Развертывание proxy-сервера

1. Подключиться к серверу `proxy`.
2. Скачать скрипт:

```bash
curl -O https://raw.githubusercontent.com/RanisAbit/otus/refs/heads/main/scripts/proxy.sh
```

3. Сделать файл исполняемым:

```bash
chmod +x proxy.sh
```

4. Запустить скрипт, передав IP-адреса backend-серверов в качестве аргументов:

```bash
./proxy.sh ip_address_backend-1 ip_address_backend-2
```

5. После завершения скрипта проверить доступность proxy-сервера:

```bash
curl -s http://127.0.0.1/health.txt
```

В ответе должны отображаться hostname backend-серверов. При повторных запросах ответ должен приходить поочередно от разных backend-серверов, что подтверждает работу балансировки.

### Установка агентов мониторинга и логирования

1. Скачать скрипт:

```bash
curl -O https://raw.githubusercontent.com/RanisAbit/otus/refs/heads/main/scripts/agents.sh
```

2. Сделать файл исполняемым:

```bash
chmod +x agents.sh
```

3. Запустить скрипт:

```bash
./agents.sh
```

4. После завершения скрипта убедиться, что node_exporter и Filebeat установлены и запущены.

## Проверка работоспособности

После завершения всех этапов необходимо убедиться, что:
- контейнер Nginx запущен;
- proxy-сервер доступен локально;
- балансировка между `backend-1` и `backend-2` работает корректно;
- node_exporter передает метрики в Prometheus;
- Filebeat отправляет логи в Logstash.

## Команды для проверки

Проверка контейнеров:

```bash
docker ps -a
```

Проверка доступности proxy-сервера:

```bash
curl -s http://127.0.0.1/health.txt
```

## Примечания

- Скрипт `proxy.sh` должен запускаться с двумя аргументами — IP-адресами `backend-1` и `backend-2`.
- Часть конфигурационных файлов и Docker Compose-описаний формируется внутри скриптов через heredoc (`EOF`).
- При изменении логики развертывания необходимо вносить изменения не только в отдельные конфигурационные файлы репозитория, но и в соответствующие скрипты.
