# Инструкция по развертыванию сервера мониторинга

Данный документ описывает порядок развертывания сервера мониторинга инфраструктуры проекта.

В результате выполнения инструкции должны быть настроены:
- Prometheus в Docker-контейнере;
- Grafana в Docker-контейнере.

## Состав сервера мониторинга

Сервер мониторинга включает в себя:
- Docker для развертывания Grafana и Prometheus.

## Используемые файлы

### Скрипты
- `scripts/monitoring.sh` — установка Grafana и Prometheus в Docker.

### Docker Compose
- `docker/grafana/docker-compose.yml`
- `docker/prometheus/docker-compose.yml`

### Конфигурационные файлы
- `configs/prometheus/prometheus.yml`
- `configs/sql-slave-server/mysqld.cnf`

## Предварительные требования

Перед началом развертывания необходимо убедиться, что:
- сервер `observ` доступен по SSH;
- на сервере используется Ubuntu Server;
- сервер имеет доступ в интернет;
- доступен пользователь `root` или пользователь с правами `sudo`.

Во всех примерах ниже предполагается выполнение команд от имени `root`.

## Таблица развертывания

| Сервер | Назначение | Используемые скрипты |
|---|---|---|
| `observ` | Monitoring + Logging | `monitoring.sh`, `logging.sh` |

## Общая схема запуска скриптов

Для каждого этапа используется одинаковый порядок действий:
1. Подключиться к серверу по SSH.
2. Скачать необходимый скрипт из репозитория с помощью `curl`.
3. Сделать файл исполняемым.
4. Запустить скрипт.
5. Проверить результат выполнения.

Если скрипт не удалось скачать автоматически, его можно создать вручную и вставить в него содержимое из репозитория.

## Порядок развертывания

### Установка сервера мониторинга (Prometheus и Grafana)

1. Скачать скрипт:

```bash
curl -O https://raw.githubusercontent.com/RanisAbit/otus/refs/heads/main/scripts/monitoring.sh
```

2. Сделать файл исполняемым:

```bash
chmod +x monitoring.sh
```

3. Запустить скрипт:

```bash
./monitoring.sh
```

4. Во время выполнения скрипта необходимо создать пароль для входа в веб консоль grafana.

5. Во время выполнения скрипта необходимо ввести IP-адреса виртуальных машин (серверов), на которых установлены агенты `node_exporter`.

## Проверка работоспособности

После завершения скрипта необходимо убедиться в следующем:
- Веб-интерфейс **Grafana** доступен по порту `3000`.
- После настройки дашбордов в Grafana **видны метрики** с целевых виртуальных машин.
- Агенты (targets) отображаются со статусом `UP` в веб-интерфейсе **Prometheus** по порту `9090` (в разделе *Status* → *Targets*).

## Примечания

- Часть конфигурационных файлов и Docker Compose-описаний формируется внутри скриптов через heredoc (`EOF`).
- При изменении логики развертывания необходимо вносить изменения не только в отдельные конфигурационные файлы репозитория, но и в соответствующие скрипты.
- Для быстрого создания дашборда в Grafana можно использовать готовый шаблон с **ID 1860**.  
На странице **Import dashboard** достаточно ввести `1860` в поле загрузки дашборда, затем нажать **Load** и выбрать источник данных Prometheus.
